# 使用记忆飞轮的最佳方式

> 从使用者角度出发，让记忆飞轮真正成为你的工作伙伴

---

## 两种使用方式

### 方式一：每次项目完成后新建对话（推荐）

```
项目A对话 → 完成 → 关掉窗口 → 新开对话 → 项目B
```

**优点**：
- 每次新对话，AI 自动拥有完整记忆
- 上下文干净，不会被之前的任务干扰
- 适合同时做多个独立项目

**记忆飞轮表现**：
- 新会话启动时，AGENTS.md 已包含所有历史记忆
- AI 知道你是谁、之前做了什么、接下来要干嘛
- 无需手动"回忆"，直接开始工作

---

### 方式二：一直聊下去

```
同一对话窗口，持续对话 1 小时、2 小时...
```

**适用场景**：
- 单一项目，持续深入
- 不想丢失当前对话的上下文

**注意事项**：
- 对话超过一定长度，上下文会被压缩
- 记忆飞轮会在后台处理，但注入的上下文会逐渐变少
- 建议：超过 1 小时的重要对话结束后，手动触发一次注入

```bash
# 手动触发记忆注入
iflow-memory inject
```

## 推荐配置

### 方式一：全部默认（最省心）

所有功能默认开启，记忆飞轮全自动工作。

适合：不想折腾、让 AI 自己管理

---

### 方式二：精简模式（省 token）

如果 token 消耗是问题，可以关掉部分功能：

| 功能 | 建议 | 原因 |
|------|------|------|
| 索引生成 | 开 | 必开，否则无法定位历史 |
| 结构化摘要 | 开 | 必开，核心功能 |
| 记忆分类 | 开 | 必开，提取结构化记忆 |
| 氛围快照 | 关 | 比较占地方，可关 |
| 每日回顾 | 开 | 记录当天工作 |
| 知识图谱 | 关 | 需要向量搜索，可关 |
| 每日简报 | 开 | 快速了解昨天 |

操作：
> "帮我关掉氛围快照和知识图谱"

---

### 方式三：极致精简

只保留核心记忆提取：

> "帮我只保留索引生成、结构化摘要和记忆分类，其他全关"

## 常见问题

### Q: 记忆飞轮占用多少 token？

当前 AGENTS.md 约 2200 tokens，在 100K 上下文中约占 2%。

可以通过关闭氛围快照、知识图谱等功能进一步减少。

---

### Q: 新会话时 AI 好像不记得之前的事？

1. 检查守护进程是否在运行：
```bash
ps aux | grep iflow_memory
```

2. 检查 AGENTS.md 是否存在：
```bash
ls -la ~/.iflow/AGENTS.md
```

3. 如果 AGENTS.md 是空的或太旧，手动触发注入：
```bash
iflow-memory inject
```

---

### Q: 怎么知道记忆飞轮在工作？

1. 查看 AGENTS.md 最后更新时间：
```bash
grep "最后更新" ~/.iflow/AGENTS.md
```

2. 查看记忆数量：
```bash
grep "记忆总数" ~/.iflow/AGENTS.md
```

---

### Q: 守护进程会一直运行吗？

会的。只要服务器不重启，守护进程会一直后台运行。

如果服务器重启，需要手动启动：
```bash
iflow-memory start
```

---

### Q: 可以同时用多个 AI 助手吗？

可以。记忆飞轮支持多渠道：

- iFlow CLI
- Claude Code
- 飞书机器人
- 自定义 MCP 客户端

每个渠道的对话都会自动被记忆，注入时按渠道隔离。

## 快速命令汇总

```bash
# 启动（服务器重启后需要）
iflow-memory start

# 查看状态
iflow-memory status

# 手动触发记忆注入
iflow-memory inject

# 查看功能列表
iflow-memory features list

# 搜索记忆（让 AI 帮你搜）
# 直接告诉 AI："帮我搜一下上次部署的事情"
```

## 最佳实践总结

1. **保持守护进程运行** —— 一次启动，长期工作
2. **每个项目新建对话** —— 上下文清晰，记忆不串
3. **让 AI 自己管理** —— 告诉它你要做什么，不用记命令
4. **需要时直接问** —— "上次我们改了什么？""记得上次那个 bug 吗？"
5. **定期检查** —— 偶尔看看 AGENTS.md 是否正常更新

---

**核心思路**：把记忆飞轮当空气——它一直在后台工作，你只需要正常和 AI 对话，它就会帮你记住一切。
